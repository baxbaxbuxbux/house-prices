# house-prices
# სახლის ფასების პროგნოზირება (House Prices - Advanced Regression Techniques)

## Kaggle-ის კონკურსის მოკლე მიმოხილვა
მოცემული პროექტი ეხება Kaggle-ის ერთ-ერთ ყველაზე პოპულარულ კონკურსს, სადაც მიზანია აიოვას შტატის ქალაქ ემსში (Ames, Iowa) საცხოვრებელი სახლების გაყიდვის ფასის პროგნოზირება. მონაცემთა ბაზა მოიცავს 79 ცვლადს, რომლებიც აღწერენ სახლის თითქმის ყველა ასპექტს — ფართობიდან დაწყებული, სახურავის მასალითა და კეთილმოწყობით დამთავრებული.

## ჩემი მიდგომა პრობლემის გადასაჭრელად
ჩემი მიდგომა დაეფუძნა მონაცემთა ევრისტიკულ ანალიზს და ექსპერიმენტულ ციკლს. პროცესი დაიწყო მონაცემთა ვიზუალიზაციითა და მნიშვნელოვანი ნიშან-თვისებების (features) შერჩევით. თავდაპირველად მოდელი აიგო მხოლოდ რიცხვით ცვლადებზე, ხოლო შემდგომ ეტაპზე დაემატა კატეგორიული მონაცემები **Residual-based Target Encoding**-ის გამოყენებით, რამაც საგრძნობლად გააუმჯობესა მოდელის სიზუსტე.

## Feature Engineering & Cleaning

### რიცხვითი ცვლადების დამუშავება
* **Nan მნიშვნელობების შევსება**: გამოვიყენე `SimpleImputer`. GridSearch-ის შედეგად საუკეთესო სტრატეგიად გამოიკვეთა `most_frequent`.
* **სკალირება**: ყველა რიცხვითი ცვლადი დასკალირდა `StandardScaler`-ით.
* **არაწრფივი დამოკიდებულებები**: ჩავატარე ექსპერიმენტი ხარისხობრივი ფუნქციების დამატებაზე (`PolynomialExpander`), თუმცა ლოგარითმულმა ანალიზმა და სლოუპების (slopes) გამოთვლამ აჩვენა, რომ მათი ჩამატება პერფორმანსს მნიშვნელოვნად არ აუმჯობესებდა, ამიტომ საბოლოო მოდელში არ გამოვიყენე.

### კატეგორიული ცვლადების დამუშავება
* **Residual-based Smooth Target Encoding**: კატეგორიული ცვლადებისთვის გამოვიყენე Target Encoding.
* **ხმაურის შემცირება**: იმისათვის, რომ ენკოდინგი ყოფილიყო უფრო ზუსტი, სამიზნე ცვლადად გამოვიყენე რეალურ ფასს გამოკლებული რიცხვითი მოდელის პროგნოზები (Residuals). ამით კატეგორიულმა მოდელმა ისწავლა მხოლოდ ის სხვაობა, რასაც წმინდა რიცხვითი ცვლადები ვერ ხსნიდნენ.

## Feature Selection
მნიშვნელოვანი ცვლადების შესარჩევად გამოვიყენე ორი ძირითადი მეტრიკა:
1. **კორელაციის მატრიცა**: წრფივი კავშირების დასადგენად.
2. **Mutual Information**: არაწრფივი დამოკიდებულებების აღმოსაჩენად.

შევქმენი `FeatureSelector` კლასი, რომელმაც მოახდინა ტოპ 10-15 ცვლადის სელექცია. საინტერესოა, რომ `MSSubClass`-მა ვიზუალური კავშირი არ აჩვენა, თუმცა მისი ამოღება მოდელის შედეგს აუარესებდა, ამიტომ ის დავტოვე.

## Training & ექსპერიმენტების ანალიზი

### ტესტირებული მოდელები და შედეგები
1. **Baseline Linear Regression (მხოლოდ რიცხვითი)**:
   - **Validation RMSE**: ~36,202
   - **Gap**: ~6,935
   - *დასკვნა*: მაღალი ცდომილება მიუთითებდა როგორც Underfitting-ზე, ისე სუსტ გენერალიზაციაზე.
2. **Linear Regression + IQR Outlier Removal**:
   - **RMSE (გაუფილტრავ ვალიდაციაზე)**: ~39,830
   - *დასკვნა*: აუთლაიერების ამოგდება IQR მეთოდით არ გამოდგა ეფექტური, რადგან დაიკარგა მნიშვნელოვანი ინფორმაცია.
3. **Combined Model (Numeric + Residual Target Encoding)**:
   - **Validation RMSE**: ~30,542
   - *დასკვნა*: ეს აღმოჩნდა საუკეთესო მიდგომა.

### საბოლოო მოდელის შერჩევის დასაბუთება
საუკეთესო მოდელმა აჩვენა ყველაზე დაბალი Validation RMSE. კატეგორიული ცვლადების "ნარჩენებზე" დატრენინგებამ მოდელს მისცა საშუალება დაეჭირა ისეთი ნიუანსები, რომლებსაც მხოლოდ სტანდარტული რიცხვითი მახასიათებლები ვერ ხსნიდა.
