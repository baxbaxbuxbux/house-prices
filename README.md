# სახლის ფასების პროგნოზირება 

## Kaggle-ის კონკურსის მოკლე მიმოხილვა
მოცემული პროექტი ეხება Kaggle-ის ერთ-ერთ  კონკურსს, სადაც მიზანია  საცხოვრებელი სახლების გაყიდვის ფასის პროგნოზირება. მონაცემთა ბაზა მოიცავს ცვლადებს, რომლებიც აღწერენ სახლის თითქმის ყველა ასპექტს — ფართობიდან დაწყებული, სახურავის მასალითა და კეთილმოწყობით დამთავრებული.

## ჩემი მიდგომა პრობლემის გადასაჭრელად
ჩემი მიდგომა დაეფუძნა მონაცემთა ანალიზს და ექსპერიმენტულ ციკლს. პროცესი დაიწყო მონაცემთა ვიზუალიზაციითა და მნიშვნელოვანი feature-ების შერჩევით. თავდაპირველად მოდელი აიგო მხოლოდ რიცხვით ცვლადებზე, ხოლო შემდგომ ეტაპზე დაემატა კატეგორიული მონაცემები **Residual-based Target Encoding**-ის გამოყენებით, რამაც გააუმჯობესა მოდელის სიზუსტე.

## Feature Engineering & Cleaning

### თავიდან გამოვაცალკევე რიცხვითი და კატეგორიული ცვლადები მათზე ცალ-ცალკე სამუშაოდ.

### რიცხვითი ცვლადების დამუშავება
* **Nan მნიშვნელობების შევსება**: გამოვიყენე `SimpleImputer`. GridSearch-ის შედეგად საუკეთესო სტრატეგიად გამოიკვეთა `most_frequent`.
* **სკალირება**: ყველა რიცხვითი ცვლადი დასკალირდა `StandardScaler`-ით.
* **ვიზუალური ანალიზი**: ყველაზე მნიშვნელოვანი feature-ებზე ფასის დამოკიდებულების გრაფიკები ავაგე, რათა უკეთ დამენახა დამოკიდებულება. აღმოჩნდა, რომ ამ feature-ებისგან ფასი ყველაზე ზრდადად იყო დამოკიდებული, გარდა MSSubClass-ისა.
* **არაწრფივი დამოკიდებულებები**: ფასის ამ feature-ებზე დამოკიდებულება მართალია ზრდადი იყო, თუმცა ზოგ შემთხვევაში წრფივი არ ყოფილა, შესაბამისად გადავწყვიტე, რომ წრფივი რეგრესიის დროს feature-ებში დამემატებინა არსებული feature-ების გარკვეული ხარისხები. იმის გასარკვევად, თუ რომელი ხარისხების დამატება იქნებოდა ყველაზე მიზანშეწონილი, ავაგე ფასის ლოგარითმის თითოეული feature-ს ლოგარითმზე დამოკიდებულების გრაფიკი, თუმცა მცირე ცვლილებით, რადგან უფრო ლოგიკური იყო y=y(x=x_avg)+k(x-x_avg)^n დამოკიდებულება, რის გასათვალისწინებლადაც y-ც და x-ც ლოგარითმის ამოღებამდე დავასკალირე და 0 თან ახლო წერტილები ამოვაგდე ცდომილების შესამცირებლად, თითოეული გრაფიკი კი ორად გაიხლიჩა, x-x_avg<0 და x-x_avg>0-ს შესაბამისად. მიღებული გრაფიკების slope-ებს ყველაზე ძლიერი დამოკიდებულების ხარისხები უნდა მოეცა, თუმცა მიღებული ხარისხების ჩამატებამ პერფორმანსი არ გააუმჯობესა, ამიტომ საბოლოო მოდელში არ გამოვიყენე.

## Feature Selection
მნიშვნელოვანი ცვლადების შესარჩევად გამოვიყენე ორი ძირითადი მეტრიკა:
1. **კორელაციის მატრიცა**: წრფივი კავშირების დასადგენად.
2. **Mutual Information**: არაწრფივი დამოკიდებულებების აღმოსაჩენად.

შევქმენი `FeatureSelector` კლასი, რომელმაც მოახდინა ტოპ 10-15 ცვლადის სელექცია. საინტერესოა, რომ `MSSubClass`-მა ვიზუალური კავშირი არ აჩვენა, თუმცა მისი ამოღება მოდელის შედეგს აუარესებდა, ამიტომ ის დავტოვე.

---

### კატეგორიული ცვლადების დამუშავება
* **Residual-based Smooth Target Encoding**: კატეგორიული ცვლადებისთვის გამოვიყენე Target Encoding.
* **ხმაურის შემცირება**: იმისათვის, რომ ენკოდინგი ყოფილიყო უფრო ზუსტი, სამიზნე ცვლადად გამოვიყენე რეალურ ფასს გამოკლებული რიცხვითი მოდელის პროგნოზები (Residuals). ამით კატეგორიულმა მოდელმა ისწავლა მხოლოდ ის სხვაობა, რასაც წმინდა რიცხვითი ცვლადები ვერ ხსნიდნენ.



## Training & ექსპერიმენტების ანალიზი

### ტესტირებული მოდელები და შედეგები
1. **Baseline Linear Regression (მხოლოდ რიცხვითი)**:
   - **Validation RMSE**: ~36202
   - **Training RMSE**: ~29266
   - **Gap**: ~6935
   - *დასკვნა*: მაღალი ცდომილება მიუთითებდა Underfitting-ზე.
2. **Linear Regression + IQR Outlier Removal**:
   - **სტრატეგია**: მეორე მეთოდისთვის გადავწყვიტე outlier-ების ამოგდება და მოდელის მიღებულ    მონაცემებზე გაწრთვნა. ამისათვის IQR მეთოდი გამოვიყენე. 
   - **RMSE (გაუფილტრავ ვალიდაციაზე)**: ~39,830
   - *დასკვნა*: აუთლაიერების ამოგდება IQR მეთოდით არ გამოდგა ეფექტური, რადგან დაიკარგა მნიშვნელოვანი ინფორმაცია, იმის მიუხედავად, რომ გაფილტრულ მონაცემებზე საკმაოდ დაბალ ცდომილებას ვიღებდით.
4. **Combined Model (Numeric + Residual Target Encoding)**:
   - **Validation RMSE**: ~30,542
   - *დასკვნა*: ეს აღმოჩნდა საუკეთესო მიდგომა.

### საბოლოო მოდელის შერჩევის დასაბუთება
საუკეთესო მოდელმა აჩვენა ყველაზე დაბალი Validation RMSE. კატეგორიული ცვლადების "ნარჩენებზე" დატრენინგებამ მოდელს მისცა საშუალება დაეჭირა ისეთი ნიუანსები, რომლებსაც მხოლოდ სტანდარტული რიცხვითი მახასიათებლები ვერ ხსნიდა.
